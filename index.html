Q4."Print output: 
            var x = 21;
            var girl = function () {
                console.log(x);
                var x = 20;
            };
            girl ();"

o/p: undefined 

REASON: The output is undefined, because of 
            i.  global scope : even though the x is declared at the top, it has global scope, which means it can be accessed anywhere except inside the function but we are having the console statement inside the 
                function. Hence it cannot be accessed.
            ii. Variable hoisting: The variable can be accessed even though it is initialized and declared after the console statement in the girl function. Here we can access only the variable but not it's value

Q7.Write a function that accepts parameter n and returns factorial of n

function factorial(n){
            let fact=1;

                fact *=i;
            }
        console.log("The factorial of "+ n+ " is "+ fact)}
        factorial(10);
        
       o/p: The factorial of 10 is 3628800

                return function(m){
                    console.log(`Hi!! ${name}, ${m}`);
                }
            }
            const greet_message = greet('EA19');
            greet_message(""Welcome To PrepBytes"")"

o/p: Hi!! EA19, Welcome To PrepBytes 
REASON: Here the greet is declared as a function with parameter 'name', and inside that greet function we are returning a function with parametr 'm'
            which consists of a console statement, and then we have declared a new constant greet_message as a function of greet with parameter "EA19", hence ${name} becomes EA19,
             and now we have called  this greet_message function with parameter "Welcome to PrepBytes",  and ${m} becomes "Welcome to PrepBytes".
